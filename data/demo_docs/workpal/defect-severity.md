# 缺陷严重等级与响应规范（样例 · 虚构）

> BizMind Demo 虚构 WorkPal 项目管理文档。

## 1. 严重等级定义

| 等级 | 定义 | 示例 |
|------|------|------|
| P0 | 生产不可用或数据丢失 | 支付失败、数据库不可写 |
| P1 | 核心功能不可用 | 登录失败、订单无法提交 |
| P2 | 重要功能受损有 workaround | 报表导出失败 |
| P3 | 轻微问题或 UI 缺陷 | 文案错误、对齐问题 |

## 2. 响应与修复时限

| 等级 | 响应时限 | 修复或 workaround 时限 |
|------|----------|--------------------------|
| P0 | **2 小时内** | **24 小时内**修复或提供 workaround |
| P1 | 4 小时内 | 3 个工作日 |
| P2 | 1 个工作日 | 下一迭代 |
| P3 | 下一迭代规划 | 按优先级排期 |

## 3. 迭代流程

1. **Backlog 梳理**：Product Owner 按价值排序
2. **Sprint Planning**：每迭代 2 周，容量按团队速率
3. **Daily Standup**：15 分钟，阻塞项当天升级
4. **Review & Retro**：迭代末演示 + 改进项

## 4. 缺陷流转

```
新建 → 确认 → 处理中 → 待验证 → 已关闭
              ↘ 拒绝（需说明原因）
```

## 5. 升级机制

- P0/P1 超过响应时限自动通知 Engineering Manager
- 跨团队阻塞在 standup 标记为 `blocked`
